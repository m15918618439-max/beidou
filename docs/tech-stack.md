# 技术栈规范

## 当前技术栈

- HTML5：页面结构、SEO、静态路由容器。
- CSS3：响应式布局、视觉样式、动效、主题变量。
- Vanilla JavaScript：页面路由、滚动动画、计数器、轮播、弹窗表单。
- 静态资源：SVG、PNG、MP4。

## 当前不引入的内容

当前官网不使用框架和构建工具：

- 不使用 React、Vue、Next.js、Nuxt 等应用框架。
- 不使用 Vite、Webpack 等构建工具。
- 不使用 TypeScript 编译链。
- 不引入包管理器锁文件。

原因是当前官网是纯静态发布场景，页面可以直接打开和部署。先把结构整理清楚，再决定是否升级技术栈。

## 何时升级技术栈

满足以下任一条件时，再考虑升级到 Vite 或框架化方案：

1. 页面数量明显增加，需要组件复用和多入口路由。
2. 需要接入后台接口、内容管理系统或表单服务。
3. 需要多人长期维护，CSS 和 JS 拆分到多个模块后仍然难以管理。
4. 需要图片压缩、缓存版本号、自动部署等构建能力。

## 升级建议

若后续升级，建议路线为：

```text
静态 HTML/CSS/JS
→ Vite + 原生模块化 JS
→ Vite + TypeScript
→ 按需要选择 React/Vue
```

不要一步到位引入复杂框架，避免官网内容维护被工程复杂度拖慢。
